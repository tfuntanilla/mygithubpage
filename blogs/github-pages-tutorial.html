Keeping gh-pages in sync with the master

01/09/2015

I have this website set up through GitHub Pages after getting my one year of free .me domain from Namecheap by being a university student. The initial set up was pretty easy to follow, and the link to GitHub pages has step by step instructions on how to do it. I have mine set up as a Project Site and started everything from scratch, but you can also use one of their pre-built themes. When I was new to this whole thing, I thought you simply had to commit the changes to the master branch and everything will automatically update by itself, but as it turned out, it was a little more involved than that. It's not a very complicated process, but it still took me at least an hour when I first tried to do updates on this website. I googled tutorials and they all pretty much tell you the same instructions, but things wouldn't work out smoothly for me for some reason... Issues happened again when I tried to update this website the second time, and the third time, and pretty much each time except once when everything worked after one commit without any problems.

So on this post, I'm going to keep note of the Git errors and issues I encounter every time I update this website. I'll post about the solutions that worked for me. Generally, updating this website or any GitHub Pages site requires at least the following step by step:

1: git checkout master // first you have to be in the master branch
2: git add . // add all the files to commit
3: git status // to see what changes are being committed and make sure there aren't any issues
4: git commit -m "Comment about your changes/commit"
5: git push origin master // push changes to master
6: git checkout gh-pages // now go to the gh-pages branch
7: git rebase master // or merge instead of rebase, whichever you prefer; this brings gh-pages up to date with master
8: git push origin gh-pages // push changes to the gh-pages branch
9: git checkout master // return to the master branch

The git commands above will sync gh-pages branch with the master branch, and if everything worked without merge error or whatever, the site will update after a few minutes. Line 7 is the command that gives me problems, like 95% of the time, and it's the most important command since it's supposed to do the syncing.
