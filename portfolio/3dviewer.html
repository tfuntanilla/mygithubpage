This is a project I completed for my computer graphics class last Fall Quarter 2014. This is probably the most time-consuming class I've ever taken, but I ended up really liking it, and I got an A in the class, so the time and effort was definitely worth it.

It's a desktop program that allows users to create a 3D scene that they can manipulate or navigate into. The scene models/objects can be loaded from an .obj or a JSON file, and the program renders the objects. The user can then transform each individual object (translate, rotate, or scale), manipulate the lighting and the type of shading (flat, Gouraud, or Phong) of the scene, and add textures for a more realistic rendering. It also allows the user to save a scene in JSON format, so it can be loaded at a later time.

Through this project, I also learned the graphics pipeline and even implemented it in the actual program. That part was definitely the most challenging aspect since it was basically implementing the same pipeline OpenGL uses to render 3D objects. The most fun part for me is implementing the lighting and textures. That was when the program got really interesting for me since I actually got to render scenes that look somewhat realistic.

Few parts in my implementation are not fully correct, so I plan to work on those when I have the time. I also plan to fix the UI since I don't quite like it right now and maybe add more advanced computer graphics concepts.

Tools used: OpenGL, C++, and Qt
